Implementation: Outlier Detection¶

Detecting outliers in the data is extremely important in the data preprocessing step of any analysis. The presence of outliers can often skew results which take into consideration these data points. There are many "rules of thumb" for what constitutes an outlier in a dataset. Here, we will use Tukey's Method for identfying outliers: An outlier step is calculated as 1.5 times the interquartile range (IQR). A data point with a feature that is beyond an outlier step outside of the IQR for that feature is considered abnormal.

In the code block below, you will need to implement the following:

  • Assign the value of the 25th percentile for the given feature to Q1. Use np.percentile for this.
  • Assign the value of the 75th percentile for the given feature to Q3. Again, use np.percentile.
  • Assign the calculation of an outlier step for the given feature to step.
  • Optionally remove data points from the dataset by adding indices to the outliers list.

NOTE: If you choose to remove any outliers, ensure that the sample data does not contain any of these points!
Once you have performed this implementation, the dataset will be stored in the variable good_data.

Question 6¶

What are the advantages to using a K-Means clustering algorithm? What are the advantages to using a Gaussian Mixture Model clustering algorithm? Given your observations about the wholesale customer data so far, which of the two algorithms will you use and why?

Answer:

K-means:

Pros:

  1. Algorithm is one of the fastest clustering algorithms available
  2. Scales well to large number of samples
  3. Given enough time K-means will always converge
  4. Can be used on large range of applications in many different fields

Cons:

  1. It has the chance of falling in local minima
  2. Responds poorly to elongated clusters, or manifolds with irregular shapes
  3. In very high-dimensional space, Euclidean distance tend to become inflated (curse of dimensionality)
  4. Have to decided how many clusters are required. Too many means separating clusters that have the same features. Too few means combining features into one cluster that are actually separate clusters.

Gaussian Mixtures:

Pros:

  1. Good for density estimation
  2. It is the fastest algorithm for learning mixture models
  3. Uses expectation-maximization to give a weighted average of the chance of a data point being in a certain cluster. It explains how point exactly in the middle of two clusters can give half weight to one cluster and half weight to the other.

Cons:

  1. Insufficient points per mixture makes estimating the covariance matrices difficult, and can cause the algorithm to diverge and find solutions with infinite likelihood unless the covariances are artificially regularized.
  2. The algorithm will always use all the components which it has access, needing held-out data or theoretical criteria to decide how many components to use in the absence of external cues.
  3. The model doesn't know which points came from which latent component.

I will be using the Gaussian Mixture model for the data, since it is more like a density problem by the way the data is organized numerically. When observing the data in scatterplot it's clear that there aren't specified clusters, so using a model that will define the clusters for me will be preferred.
